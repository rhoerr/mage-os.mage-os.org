# Mage-OS Website User Personas

This document defines 8 user personas for UX assessment of the Mage-OS website. Each persona has calibrated skepticism levels, typical entry points, and specific tasks for mock UX sessions.

---

## Persona 1: New Merchant Evaluating Platforms

### Profile

| Attribute | Description |
|-----------|-------------|
| **Name** | Sarah Chen |
| **Role** | Business owner launching first e-commerce store |
| **Company Size** | Solo entrepreneur with 1-2 contractors |
| **Tech Level** | Low-Medium (comfortable with SaaS tools, limited technical knowledge) |
| **Skepticism Level** | Medium |
| **Entry Point** | Google search: "open source ecommerce platform" or "Magento alternative" |

### Background
Sarah runs a successful wholesale business and wants to expand into direct-to-consumer online sales. She's evaluated Shopify but wants more control and lower ongoing costs. She's heard Magento is powerful but complex.

### Goals
- Understand if Mage-OS is suitable for a non-technical person
- Assess total cost of ownership (hosting, development, maintenance)
- Find out how to get started without heavy technical investment
- Evaluate if it can scale as her business grows

### Pain Points
- Overwhelmed by technical jargon
- Worried about ongoing maintenance and security
- Needs to understand what help she'll need to hire

### Decision Factors
- Ease of getting started
- Clear cost expectations
- Available support options
- Success stories from similar merchants

### UX Session Tasks
1. Understand what Mage-OS is and how it differs from competitors
2. Find out what technical requirements are needed
3. Determine what ongoing costs and resources are required
4. Locate options for getting professional help with setup
5. Assess whether Mage-OS is appropriate for her skill level

## Persona 2: Existing Magento User

### Profile

| Attribute | Description |
|-----------|-------------|
| **Name** | Marcus Weber |
| **Role** | E-commerce Director at established online retailer |
| **Company Size** | 50-200 employees, $10M+ annual revenue |
| **Tech Level** | Medium (understands e-commerce, works with dev team) |
| **Skepticism Level** | Medium-High |
| **Entry Point** | Magento community forum link or Meet Magento event |

### Background
Marcus manages a Magento Open Source store that's been running for 6+ years. He's heard about Mage-OS at a Meet Magento conference and is curious but cautious. His store has custom modules and integrations.

### Goals
- Understand the specific benefits of migrating to Mage-OS
- Assess risk and effort of migration
- Verify compatibility with existing extensions and customizations
- Understand the governance and long-term viability of Mage-OS

### Pain Points
- Slow security patch releases from Adobe
- Concerns about Adobe's commitment to open source
- Tired of waiting for community PRs to be merged
- Worried about platform stability during any migration

### Decision Factors
- Migration complexity and time
- Extension and theme compatibility
- Community stability and governance
- Ability to switch back if needed

### UX Session Tasks
1. Find specific differences between Mage-OS and Magento Open Source
2. Locate migration guide and assess complexity
3. Verify extension compatibility claims
4. Understand governance structure and long-term sustainability
5. Find rollback procedures if migration fails

## Persona 3: Developer/Contributor

### Profile

| Attribute | Description |
|-----------|-------------|
| **Name** | Alexei Petrov |
| **Role** | Senior Magento Developer at an agency |
| **Company Size** | 30-person Magento-focused agency |
| **Tech Level** | High (PHP expert, Magento certified) |
| **Skepticism Level** | Low-Medium |
| **Entry Point** | GitHub discovery or technical blog post |

### Background
Alexei has 8 years of Magento experience and is frustrated with Adobe's slow PR review process. He's contributed to Magento core and wants to contribute to a project that values developer input.

### Goals
- Understand the technical architecture and contribution process
- Find the best way to contribute code or documentation
- Evaluate the quality of the codebase and community practices
- Connect with other active contributors

### Pain Points
- PRs sitting unreviewed for months on Magento
- Limited influence on platform direction
- Wants to work on meaningful improvements
- Needs documentation for advanced customizations

### Decision Factors
- Contribution process efficiency
- Code quality and standards
- Community activity and responsiveness
- Technical documentation quality

### UX Session Tasks
1. Find the GitHub repository and contribution guidelines
2. Understand the review and merge process for PRs
3. Locate technical documentation (DevDocs)
4. Find information about weekly tech meetings
5. Assess community activity (GitHub issues, Discord, etc.)

## Persona 4: Business Decision-Maker

### Profile

| Attribute | Description |
|-----------|-------------|
| **Name** | Jennifer Brooks |
| **Role** | CTO at mid-size e-commerce company |
| **Company Size** | 200-500 employees, $50M+ revenue |
| **Tech Level** | Low-Medium (strategic tech knowledge, not hands-on) |
| **Skepticism Level** | High |
| **Entry Point** | Industry report, analyst recommendation, or board question |

### Background
Jennifer oversees technology strategy for a multi-brand retailer. The board is questioning their Adobe Commerce investment and asking about alternatives. She needs to assess Mage-OS as a potential cost-saving option.

### Goals
- Understand total cost of ownership vs Adobe Commerce
- Assess organizational credibility and stability
- Evaluate risk factors and mitigation strategies
- Find enterprise success stories and references

### Pain Points
- High Adobe Commerce licensing costs
- Needs to justify platform decisions to the board
- Concerned about vendor lock-in
- Requires enterprise-level support guarantees

### Decision Factors
- Financial viability and TCO
- Organizational governance and stability
- Risk assessment and mitigation
- Available enterprise support options

### UX Session Tasks
1. Find clear comparison with Adobe Commerce (features and costs)
2. Assess the organization's governance and financial sustainability
3. Locate enterprise case studies or references
4. Understand what professional support options exist
5. Evaluate the risk of depending on a community-driven project

## Evaluation Criteria

Each persona session will rate pages on a 1-5 scale:

| Criterion | What It Measures |
|-----------|-----------------|
| **Messaging Clarity** | Is the value proposition immediately clear for this persona? |
| **Value Proposition Strength** | Is the differentiation compelling for their specific needs? |
| **Trust Signals** | Are there adequate credibility markers (testimonials, stats, governance)? |
| **CTA Effectiveness** | Are next steps clear and appropriate for this persona? |
| **Information Findability** | Can they find what they need quickly? |

**Scale:**
- 1 = Poor / Missing
- 2 = Weak / Confusing
- 3 = Adequate
- 4 = Good
- 5 = Excellent
